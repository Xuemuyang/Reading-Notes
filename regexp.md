# JS正则表达式

正则表达式是匹配模式，要么匹配字符，要么匹配位置

## 速查

code|detail|简介
---|---|---
`\d`|`[0-9]`|digit
`\D`|`[^0-9]`|除数字外的任意字符
`\w`|`[0-9a-zA-Z_]`|word，单词字符
`\W`|`[^0-9a-zA-Z_]`|非word
`\s`|`[ \t\v\n\r\f]`|space
`\S`|`[^ \t\v\n\r\f]`|非space
`.`|`[^\n\r\u2028\u2029]`|通配符

量词|detail|简介
`{m,}`|-|至少出现 m 次
`{m}`|`{m,m}`|出现 m 次
`?`|`{0,1}`|出现或者不出现
`+`|`{1,}`|至少出现一次
`*`|`{0,}`|出现任意次

量词加 `?` 表示贪婪

```js
{m,n}?
{m,}?
??
+?
*?
```

`|` 管道符表示任何之一 `(p1|p2|p3)`

## 字符匹配

模糊匹配：

- 横向模糊 字符串长度不固定
- 纵向模糊 某一位可以有多种可能

字符组 `[]`，one of，连字符 `-` 用来做简写，字符组第一位 `^` 表示反。

`[123456abcdefGHIJKLM]` 简写 `[1-6a-fG-M]`。

`[^abc]` 表示除 "a"、"b"、"c" 之外的任意一个字符。

`[\d\D]`、`[\w\W]`、`[\s\S]` 和 `[^]` 中的任意一个都可以表示任意字符。

### 重复

量词后面加问号实现非贪婪匹配，默认贪婪，尽可能匹配多的。

```js
{m,n}?
{m,}?
??
+?
*?
```

### 分支

一个模式可以实现横向和纵向模糊匹配，多选分支可以支持多个子模式任选其一。

分支结构与 `||` 操作符类似，短路

```js
var regex = /goodbye|good/g;
var string = "goodbye";
console.log( string.match(regex) );
// => ["goodbye"]
```
